# Лео Считалкин 🦊

Математика для второклассника: сложение и вычитание до 100, порядок действий,
таблица умножения и деление. PWA — ставится на телефон и работает без
интернета. Весь прогресс хранится только на устройстве, ничего не уходит в сеть.

**Живая версия:** https://leo-schitalkin.vercel.app

---

## Запуск

```bash
npm install
npm run dev          # http://localhost:5173
npm run build        # → dist/
npm run icons        # перерисовать иконки PWA из мордочки Лео
npm run icons:emoji  # дозакачать Twemoji для новых эмодзи из кода
```

## Деплой

`vite.config.js` использует `base: './'`, поэтому **одна и та же сборка** работает
и в корне домена (Vercel), и в подпапке `/<repo>/` (GitHub Pages) — без
пересборки и без подстановки имени репозитория.

- **Vercel** — `vercel.json` уже настроен, автодеплой при push в `main`.
- **GitHub Pages** — `.github/workflows/deploy.yml`, автодеплой при push в `main`.
  Включите Pages → Source: **GitHub Actions**.

Роутинг на `HashRouter`, так что серверные rewrites не нужны нигде.

## Что внутри

**8 разделов, 34 урока.** Разделы короткие (2–5 уроков) намеренно: раздел из
шестнадцати уроков — это месяц долбёжки одного и того же баннера без ощущения,
что куда-то пришёл.

| # | Раздел | Хозяин |
|---|---|---|
| 1 | Числа до 100 | 🦊 Лео |
| 2 | Плюс и минус | 🦊 Лео |
| 3 | Столбик и скобки | 🐆 Пятныш |
| 4 | Что такое умножение | 🐆 Пятныш |
| 5 | Таблица дальше (×3, ×4, ×6) | 🐅 Тиг |
| 6 | Самые трудные (×9, ×7, ×8) | 🐅 Тиг |
| 7 | Деление | 🐆 Пятныш |
| 8 | Всё вместе + Блиц | 🐅 Тиг |

Каждый раздел заканчивается **Повторением** — перемешанные вопросы из всех его
уроков.

### Типы заданий

Числовая прямая · блоки-десятки · сравнение (в т.ч. `35+17 ⬜ 29+21`) ·
пропущенное число · цепочки `8 → +5 → ⬜ → −3 → ⬜` · задачи с картинками ·
математический язык («найди разность», «уменьшаемое») · столбик с переносом ·
пропуск в слагаемом · скобки и порядок действий · задачи в 2 действия ·
массивы · таблица умножения · деление · соедини пары · блиц.

### Проверки и прыжки

- **«Знаю это!»** — экзамен по разделу: 12 вопросов, 3 ошибки. Сдал — раздел
  открыт целиком. Звёзды по результату: 0 ошибок → ⭐⭐⭐, 1 → ⭐⭐, 2 → ⭐.
- **«Сразу сюда»** — прыжок в закрытый раздел. Проверяет **все разделы по
  пути**, а не только цель: иначе ребёнок попадёт на умножение, не умея
  складывать. 16 вопросов, 3 ошибки.

## Как это устроено

```
src/
├── components/   Cub (3 персонажа на одном скелете), Mascot, Teacher, Icon,
│                 UiIcon, GoalRing, Sheet, NotifyGuide, Confetti, ui
├── exercises/    по одному файлу на тип задания
├── game/         curriculum, generators, store, audio, notify, messages,
│                 confetti, stickers
├── screens/      Home (карта), Lesson, LessonComplete, Collection, Settings,
│                 Tutor, PickBuddy
├── styles/       tokens.css (палитра, размеры), global.css
└── sw.js         service worker (напоминания + офлайн-кэш)
```

**Контент** — `game/curriculum.js`. Вопросы генерируются на лету в
`game/generators.js`. Добавить урок = запись в curriculum + ветка в
`buildLesson`.

**Прогресс** — `game/store.jsx`, React Context + reducer, в `localStorage` под
ключом `leo-schitalkin/v1`.

**Звук** синтезируется через WebAudio (`game/audio.js`) — ноль байт загрузки,
мгновенный отклик, работает офлайн. Озвучка заданий — Web Speech API.

**Шрифт** Nunito (SIL OFL), подмножества latin + cyrillic, 60 КБ.

**Иконки** — двух видов, и это осознанно:
- *Контент* (уроки, наклейки, предметы в задачах) — цветной **Twemoji**
  (CC-BY 4.0). Ребёнок сканирует карту глазами: красное яблоко узнаётся
  мгновенно, 34 серых контура сольются.
- *Интерфейс* (шестерёнка, звук, замок, стрелки) — **Material Symbols Rounded**
  (Apache 2.0), наследуют `currentColor`. Замок серый на сером узле, а не
  ярко-жёлтый, притягивающий взгляд к тому, что нельзя нажать.

### Три персонажа

`Cub.jsx` — один скелет, три вида. Отличаются ушами, хвостом, окрасом и
глазами; все анимации общие. Роли разведены:

- **Друг** — выбранный ребёнком, идёт по карте и радуется победам.
- **Учитель** — хозяин раздела, ведёт уроки внутри своей темы.

Состояния: `idle`, `wave`, `happy`, `cheer`, `starry`, `think`, `sleepy`.
Последние два лица растут вместе с серией: 5 подряд → happy, 10 → cheer,
20 → starry.

## Педагогика, зашитая в код

- Порядок таблиц — **2, 10, 5, 3, 4, 6, 9, 7, 8**, а не по возрастанию: каждая
  следующая опирается на уже освоенную (×4 — это ×2 дважды, ×9 — это ×10 минус
  число). ×7 и ×8 в конце: к тому моменту почти всё в них уже выучено как
  перестановки.
- **Точность считается по первой попытке.** Правильный ответ со второй даёт XP,
  но не влияет на звёзды — иначе метрика «что даётся тяжело» врёт.
- **Ошибки запоминаются** (`state.mistakes`) и возвращаются в «Работе над
  ошибками». Решил верно с первой попытки — пример выбывает.
- «Смешанная тренировка» и «Блиц» тянут факты с **весом по ошибкам**: то, что
  не получается, встречается чаще.
- Ошибка **никогда не заканчивает урок**. Сердечки только показывают старание.
  Цвет ошибки — тёплый оранжевый, звук мягкий, персонаж озадаченный, но добрый.
- **Серия правильных ответов не сбрасывается** в конце урока — она принадлежит
  ребёнку, а не уроку.

## Напоминания

Честно про предел: **уведомления при полностью закрытом приложении требуют
сервера**, а бэкенда здесь нет. Что работает:

| Механизм | Когда |
|---|---|
| Periodic Background Sync | Android Chrome, приложение установлено на экран |
| Проверка при открытии | всегда |
| Напоминание в фоне | если приложение свёрнуто, но не выгружено |

На iOS Periodic Sync не существует — остаются только два последних пункта.

22 варианта текста в `game/messages.js` на четыре ситуации. Тон намеренно **не
как у Дуолинго**: их пассивная агрессия работает на взрослых как шутка, а
семилетним читается как «приложение меня ругает».

Ограничители: не больше одного в день, никогда с 21:00 до 9:00, не приходит
если сегодня уже занимались.

Service worker не видит `localStorage`, поэтому приложение зеркалит четыре поля
в Cache API — их читают обе стороны.

## Дев-инструменты (`scripts/`)

```bash
node scripts/shot.mjs "#/" home 430 932 1500              # скриншот экрана
node scripts/shot.mjs "#/" home 430 932 1500 ".btn" seed.json  # + клики и профиль
node scripts/play.mjs numberline done 0 seed.json         # пройти урок целиком
node scripts/play.mjs exam-u2 ex "exactly:2"              # ровно 2 ошибки
node scripts/variety.mjs numberline word                  # замерить повторяемость
node scripts/probe.mjs "#/" "document.title"              # выполнить JS на странице
```

`play.mjs` реально проходит урок и проверяет XP, звёзды, наклейки, точность,
серию и список ошибок. Этим найдены, например: потерянные нажатия при быстром
тапе, экзамен, который нельзя было провалить, и вопросы, которых **не было на
экране** — скрипт «решал» примеры, которых не нарисовано, потому что читал ответ
из состояния. Теперь он этого не делает: `audit` проверяет, что вопрос виден.

Опирается на `window.__leoQ`, который есть только в dev и вырезается из
production-сборки.

## Родительская страница

`/#/tutor` — шестерёнка в углу → «Успехи». Точность по темам, флаг «тяжело
даётся» (ниже 60% при ≥4 ответах), минуты за 2 недели, сетка всех фактов
таблицы умножения по цветам, лучшая серия.

Уроки, сданные экзаменом, помечены `viaExam` — для родителя «сдал проверкой» и
«прошёл уроки» это разные вещи.
